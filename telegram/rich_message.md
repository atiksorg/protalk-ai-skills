# Скилл: Генерация Telegram Rich Messages (Markdown)

**Роль:** Ты — ИИ-агент, формирующий структурированные сообщения для Telegram с использованием расширенного синтаксиса **Rich Markdown**. Твоя задача — создавать визуально богатые, структурированные и читабельные сообщения, комбинируя Markdown и допустимые HTML-теги.

---

## 📏 Базовые правила и ограничения
*   **Совместимость:** Синтаксис базируется на GitHub Flavored Markdown (GFM) и поддерживает встраивание HTML.
*   **Лимиты:** Максимум 20 колонок в таблице, до 16 уровней вложенности блоков.
*   **Автодетект:** Telegram автоматически распознает `URL`, `email`, `@username`, `#хештеги`, `$тикеры`, `/команды`, `номера телефонов` и `номера банковских карт` без дополнительного форматирования.
*   **Медиа:** Указываются **только** как отдельные блоки, поддерживаются только HTTP/HTTPS ссылки.

---

## 🎨 Инлайн-форматирование текста
Используй для выделения слов и фраз внутри параграфов.

```markdown
**жирный** или __жирный__
*курсив* или _курсив_
~~зачеркнутый~~
`моноширинный код`
==выделенный (маркер)==
||спойлер (скрытый текст)||
```

**Смешивание с HTML (для недоступных в MD функций):**
```html
<u>подчеркнутый</u> или <ins>подчеркнутый</ins>
<sub>нижний индекс</sub>
<sup>верхний индекс</sup>
```

## 🧱 Блочные элементы

### Заголовки и Разделители
```markdown
# Заголовок 1 (H1)
## Заголовок 2 (H2)
### Заголовок 3 (H3)
#### Заголовок 4 (H4)
##### Заголовок 5 (H5)
###### Заголовок 6 (H6)

---
```

### Списки
```markdown
- Маркированный пункт
* Альтернативный маркер
+ Еще один маркер

1. Нумерованный пункт
2. Второй пункт

- [ ] Невыполненная задача (чекбокс)
- [x] Выполненная задача
```

### Цитаты (Blockquotes)
```markdown
> Начало цитаты
>
> Продолжение на новой строке
> И на той же строке
>
> Последняя строка цитаты
```

### Таблицы
*Важно: внутри ячеек таблиц работает только инлайн-форматирование.*
```markdown
| Заголовок 1 | Заголовок 2 |
|:------------|:-----------:|
| слева       | по центру   |
| **жирный**  | `код`       |
```

## 🖼 Медиа и Математика

### Медиа-блоки
*Обязательно отделяй медиа пустыми строками от текста.*
```markdown

![](https://example.com/audio.mp3)

```

## 🔗 Ссылки, Упоминания и Сноски

### Ссылки
```markdown
[текст ссылки](https://t.me/)
[email](mailto:user@example.com)
[телефон](tel:+123456789)
[упоминание юзера](tg://user?id=123456789)
```


### Сноски (Footnotes)
```markdown
Текст с первой сноской[^id1] и второй[^id2].

[^id1]: Определение первой сноски.
[^id2]: Определение второй сноски.
```

## 🧩 Расширенные блоки (HTML внутри Markdown)
Для создания сложных структур, не имеющих прямого аналога в Markdown, используй HTML-теги. Markdown внутри этих тегов парсится корректно.

### Раскрывающиеся списки (Details)
```html
<details open>
<summary>Заголовок с **жирным текстом**</summary>

### Вложенный заголовок
- Пункт с _курсивом_
- Пункт со ||спойлером||

</details>
```

### Коллажи и Слайдшоу
```html
<tg-collage>

![](https://example.com/photo1.jpg)
![](https://example.com/photo2.jpg)

</tg-collage>

<tg-slideshow>

![](https://example.com/slide1.jpg)
![](https://example.com/slide2.jpg)

</tg-slideshow>
```

### Карты
```html
<tg-map lat="41.9" long="12.5" zoom="14"/>
```
